Explanation below for the tables and the user stories.

Table: Users This table stores information about users who can access the system. Each user has a unique UserID, a Username, and a hashed Password for security. The RoleID indicates the role assigned to the user, determining their permissions within the system.

Table: Roles The Roles table defines the different roles that users can have. Each role is identified by a RoleID and has a RoleName. Roles help manage access control and permissions.

Table: Visitors The Visitors table maintains records of visitors who come to the organization. It includes a unique VisitorID, the Name of the visitor, and their Contact details.

Table: Visits The Visits table holds information about each visit made by a visitor. It includes a VisitID, associated VisitorID, Check-In and Check-Out times, and any FeedbackText provided by the visitor. This table enables tracking the duration of visits and collecting feedback.

Table: Blacklist The Blacklist table stores data about visitors who have been blacklisted. Each entry has a unique BlacklistID, associated VisitorID, and a BlacklistReason describing the reason for blacklisting. This table helps in managing visitor restrictions.

Table: Hosts The Hosts table maintains records of hosts within the organization. Each host is identified by a HostID, has a Name, and is associated with a CalendarID.

Table: Calendar The Calendar table stores booking information for host calendars. It includes a CalendarID, associated HostID, and BookedDate. This table allows receptionists to schedule host availability for greeting visitors.

APIs/User Stories below.

CreateAdminUser: The Users table stores admin user information, allowing the creation of new admin users.

ConfigureSystemSettings: Admins can access the Roles table to manage different roles and their permissions.

ManageUserAccounts: Admins use the Users and Roles tables to create, modify, and deactivate user accounts with specific roles and permissions.

PreRegisterVisitor: The Visitors table holds visitor data, including those who pre-register online.

OnSiteVisitorRegistration: Visitor details captured during on-site registration are stored in the Visitors table.

VisitorCheckIn: Visit information is recorded in the Visits table upon visitor check-in.

VisitorCheckOut: Check-out times are updated in the Visits table.

NotifyHost: Host notifications are triggered based on visit records in the Visits table.

GenerateVisitorAnalyticsReport: Admins can analyze visitor traffic, peak hours, and visit purposes using data from the Visits table.

ManageBlacklist: Data in the Blacklist table helps in managing visitors who are blacklisted.

BookHostCalendar: Receptionists use the Calendar table to book host calendars for important visitors.

ProvideFeedback: Feedback from visitors is stored in the Visits table.
